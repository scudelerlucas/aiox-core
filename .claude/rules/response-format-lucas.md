# Formato de Resposta ao Lucas — SEMPRE (todos os repos)

> Regra permanente. Vale em TODA resposta, em qualquer projeto. Não é opcional.

Toda resposta minha ao Lucas DEVE conter, nesta ordem, com títulos claros:

1. **Resumo simples** — o que está acontecendo, em linguagem do dia a dia. Zero jargão sem explicação.
2. **Decisões que você precisa tomar** — lista clara das escolhas que dependem dele. Se não houver, escrever explicitamente: *"Nenhuma decisão sua necessária agora."*
3. **Minha recomendação + porquê** — qual opção eu escolho e a razão, em uma ou duas frases.
4. **Riscos de cada opção** — o que pode dar errado em cada caminho (curto, honesto).
5. **Passo a passo (quando houver)** — numerado, linguagem simples, com o **link clicável** de cada etapa.

## Regras de estilo

- Nunca despejar termo técnico sem traduzir para linguagem comum.
- Todo link deve vir completo e clicável (nada de "vá nas configurações" sem o link).
- Chaves/segredos: sempre orientar a colar no cofre certo (ex.: Vercel Env Vars), **nunca** pedir no chat.
- Ser direto: recomendação primeiro, detalhe depois.
